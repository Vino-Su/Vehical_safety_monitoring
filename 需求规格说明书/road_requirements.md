1. 模块概述
1.1 模块定位
开放道路资源管理模块是智能网联汽车安全监测平台的三大核心业务模块之一，定位为智能网联汽车道路测试与示范应用的基础资源保障模块。该模块对全市范围内符合智能网联汽车测试与示范应用条件的城市道路进行统一资源化、标准化、可视化管理，构建“资源目录化、管理精细化、查询便捷化”的开放道路资源管理体系。
1.2 核心价值
• 解决痛点：针对襄阳市智能网联开放道路资源分散、管理缺乏标准化的现状
• 核心目标：为测试示范准入审批、路线规划、过程监管提供精准、动态的道路资源支撑
• 业务闭环：与测试示范监管模块、监测数据分析模块形成联动互补，道路资源数据为测试审批提供合规基础设施依据，分析结果反哺道路资源动态调整
1.3 适用范围
本模块面向以下用户角色：
• 政府监管部门（经信、交通、公安）：进行开放道路的规划、审批与管控
• 产业运营主体（主机厂、运营企业）：查询可用的开放道路资源，辅助测试路线申请
• 第三方服务机构：获取道路资源信息以支撑技术评估

2.1.1.2 道路资源动态管控
需求描述：
支持开放道路目录的全生命周期管理，包括状态的动态调整与准入规则的精细化配置。状态变更时需进行业务校验。
功能要点：
| 序号 | 功能点 | 描述 |
|:---|:---|:---|
| 1 | 状态管理 | 支持开放道路的启用、暂停、关闭等状态变更操作 |
| 2 | 动态调整触发 | 可依据道路施工、交通管制、基础设施升级等场景动态调整道路开放状态 |
| 3 | 状态变更校验 | 修改道路状态为"暂停"或"关闭"时，系统自动校验当前是否有正在执行中的测试任务涉及该道路，若有则弹出提示窗口，列出相关测试任务清单（任务编号、测试主体、车辆信息等），并告知用户变更状态后将自动终止这些测试任务 |
| 4 | 测试任务终止 | 确认状态变更后，系统自动向测试示范监管模块发送任务终止指令，相关测试任务标记为"因道路状态变更而终止" |
| 5 | 通行管控配置 | 支持对道路测试适用标准、通行管控要求、允许运行时段（粒度到小时）进行精细化编辑 |
| 6 | 差异化准入配置 | 针对L2/L3/L4等不同自动驾驶等级配置差异化准入规则；针对载人、载货、环卫作业等不同测试场景配置差异化道路开放权限 |
| 7 | 变更记录留痕 | 所有状态变更操作均记录操作人、操作时间、变更原因、变更前后状态 |
| 8 | 道路详情查看 | 支持点击操作列的"详情"按钮，跳转至道路详情弹窗，展示道路基础标识信息、开放管控信息、基础设施信息等 |
| 9 | 关联测试任务查看 | 支持点击"关联测试任务"列的数字，打开测试任务列表弹窗，展示任务编号、测试主体、车辆信息、任务状态、时间等 |
| 10 | 测试任务详情查看 | 在关联测试任务列表弹窗中，支持点击操作列的"详情"按钮，跳转至测试任务详情弹窗，展示任务编号、任务名称、测试主体、车牌号、VIN码、自动驾驶等级、累计测试里程、测试路线、测试驾驶员、联系电话、开始时间、预计结束时间等 |

业务规则：
• 暂停/关闭状态的道路应在测试示范准入申请的道路选择列表中自动过滤，不可选择
• 状态变更原因为必填项
• 差异化准入规则应与测试示范监管模块的审批流程联动
2.1.2 道路资源可视化
需求描述：
基于高德地图构建可视化展示体系，以空间图层形式呈现道路资源图谱。支持按开放状态筛选并以不同颜色区分，支持多维度组合筛选与精准查询。
地图选型：当前阶段基于Canvas自主绘制地图实现道路资源可视化交互原型，后续对接高德地图JS API以获得真实地理底图与路径匹配能力。
功能要点：
| 序号 | 功能点 | 描述 |
|:---|:---|:---|
| 1 | 地图基础展示 | 以高德地图为底图，以空间图层形式叠加展示全量开放道路分布及路段边界 |
| 2 | 开放状态颜色区分 | 按开放状态以不同颜色渲染道路线段：<br>• 开放：绿色<br>• 暂停：橙色<br>• 关闭：红色 |
| 3 | 开放状态筛选 | 支持按开放状态进行筛选（全选/开放/暂停/关闭），地图仅显示选中状态对应的道路 |
| 4 | 基础设施展示 | 展示路侧基础设施配置信息（RSU、感知设备位置与覆盖范围），以图标叠加方式呈现 |
| 5 | 道路属性详情 | 点击道路可弹出信息窗展示道路属性详情（等级、流量、适用场景、开放时段、开放状态等） |
| 6 | 评估专题展示 | 支持切换展示道路评估模型结果专题图（复杂度、风险、场景适配、开放等级），以不同颜色梯度渲染 |
| 7 | 多维度筛选 | 支持按行政区域、测试场景、路侧基础设施覆盖程度等维度组合筛选 |
| 8 | 精准查询 | 支持按道路名称、道路编号等进行模糊/精准查询，查询结果列表展示并支持地图定位 |
| 9 | 图层控制 | 支持不同图层（基础道路层、设施层、评估结果层等）的显示/隐藏切换 |
| 10 | 图例展示 | 显示当前颜色图例说明（状态颜色、等级颜色等） |

交互要求（Web管理端）：
• 选择测试场景筛选条件后，地图自动过滤展示匹配的道路
• 选中目标道路后，地图视角自动聚焦并高亮显示该道路
• 筛选条件变更时，地图与统计列表联动刷新

3. 非功能需求
3.1 性能需求
• 地图加载与渲染：图层切换响应时间 ≤ 2秒
• 批量导入：单次支持至少500条道路数据的导入校验，校验完成时间 ≤ 10秒
• 评估计算：单条道路评估计算完成时间 ≤ 30秒
• 统计查询：常规统计查询响应时间 ≤ 3秒
3.2 数据需求
• 道路资源数据需支持与高精地图数据的对接与匹配（路径匹配/Map Matching）
• 评估模型参数需支持动态配置与版本管理
• 评估日志需支持至少2年的在线存储
• 测试示范应用频次数据需从测试示范监管模块实时同步
3.3 接口需求
• 需对接高德地图服务（JS API / Web服务API），获取地图底图、地理编码、路径匹配等能力
• 需对接路侧基础设施管理系统，获取RSU/感知设备配置数据（如有）
• 需向测试示范监管模块提供开放道路资源查询接口（支持道路列表查询、单条道路详情查询、道路可用性校验）
• 需接收测试示范监管模块的测试任务状态实时同步
3.4 安全需求
• 道路资源数据的编辑操作需进行权限控制，仅授权监管人员可编辑，第三方服务机构及产业运营主体仅具有查看权限

• 所有数据操作需记录操作日志（操作人、操作时间、IP地址、操作内容）

1. 模块概述
1.1 模块定位
测试示范监管模块是智能网联汽车安全监测平台的核心业务模块之一，针对襄阳市智能网联汽车、多功能无人小车在测试示范过程中存在的过程监管缺失、合规性难以保障、安全风险防控不足等现状，构建覆盖事前准入、事中监控、事后评价的全流程数字化监管体系。
1.2 核心价值
• 实现测试示范全主体的资质审核与信息建档
• 提供线上化、标准化的申请审批通道，替代传统线下模式
• 实现车辆运行状态的实时监控与异常自动告警
• 构建测试评价体系，为测试主体与车辆提供量化评估
• 与开放道路资源管理模块、监测数据分析模块联动，形成完整业务闭环
1.3 适用范围
• 政府监管部门（经信、交通、公安）：进行测试示范的审批、监管、评估与资格终止
• 第三方服务机构：进行申请材料的初审与技术评估
• 产业运营主体（主机厂、运营企业）：提交测试示范申请、查看审批进度、上报事故、接收通知
• 安全员/测试驾驶员：相关信息备案，参与测试过程
---
1. 功能需求
2.1 准入信息管理
构建测试示范全主体、全要素的准入信息标准化管理体系，实现企业与人员资质的电子化建档与动态更新。
2.1.1 企业信息登记管理
• 测试主体自行填写企业基本信息并提交（字段详见6.4-4）
• 监管部门/第三方机构审核备案
2.1.2 车辆信息登记管理
• 测试主体登记计划申请道路测试和示范应用的车辆基本信息（字段详见6.4-5）
• 支持车辆唯一身份标识管理
2.1.3 人员信息登记管理
• 登记测试驾驶员/安全员的姓名、身份证号、驾照经历、交通事故记录、培训情况等信息
• 安全员信息与测试主体关联，资质需审核通过（注：本次不考虑安全员排班信息）
---
2.2 测试示范准入申请
提供道路测试、示范应用、示范运营的申请通道，支持初次申请、续期、变更、新增相同配置车辆等子类型。所有申请必传材料不完整时系统阻止提交。
2.2.1 道路测试准入申请
• 包含初次申请、续期（需导入上阶段测试报告）、变更（需提交变更说明）、新增相同配置车辆（需提交必要性说明）等子功能
• 支持在线填写申请书、材料电子化上传、预览和补正
2.2.2 示范应用准入申请
• 流程与道路测试类似，材料清单按示范应用要求（含载人险、里程证明等）
• 同样支持初次、续期、变更、新增车辆
2.2.3 示范运营准入申请
• 流程同上，材料增加运营资质、风险方案、远程监控说明等
• 支持初次、续期、变更、新增车辆
2.2.4 业务申请管理
• 测试主体可提交阶段性/总结报告，查看历史申请状态，对不予受理的申请重新申办
• 支持测试报告在线查看与下载
---
2.3 测试示范准入审批管理
2.3.1 审批流程（串联审批）
审批采用严格串联流程：
1. 第三方服务机构初审：对申请材料的完整性、合规性进行审核，填写审核意见，通过后提交联席小组；不通过则退回申请人。
2. 联席小组审批（政府端）：对安全性自我声明进行确认及终审。审批通过后系统自动通知市交管部门允许发放临时牌照；不通过则退回第三方机构重新审核或直接退回申请人。
3. 临牌信息回传：市交管部门发放临时牌照后，由工作人员在系统内人工录入牌照号码、有效期等信息，回传至平台并关联对应车辆。
2.3.2 资格终止管理
联席工作小组可对已批准的测试或示范活动执行资格终止，除原方案列出的条件外，增加以下触发条件：
• 道路开放状态由开放变更为不开放（暂停/关闭）：当车辆所在测试路线关联的开放道路被暂停或关闭时，该道路上的测试任务将被自动终止（由开放道路资源管理模块联动触发）。
其他原有终止条件（车辆与声明不符、号牌到期/撤销、重大安全风险、严重违法、严重事故等）保持不变。终止操作需录入原因并记录备案，同时通知公安交警局收回临时号牌。
2.3.3 报告管理（政府端）
• 查询企业提交的各类报告，向上级部门报告测试示范情况，对专家评审材料进行归档
2.3.4 在线反馈管理
• 第三方服务机构可对企业提交的反馈进行受理、回复与闭环跟踪
---
2.4 测试示范审批配置
• 测试示范类型配置：可对道路测试、示范应用、示范运营等类型进行增删改，配置对应的审批流程节点
• 申请材料配置：支持材料目录的维护、版本管理（历史版本只读）、材料与业务类型的关联关系配置
---
2.5 车辆状态监控（含监控一张图）
2.5.1 监控一张图（核心新增）
功能定位：
监控一张图是车辆状态监控的统一可视化入口，以电子地图为核心，将车辆的位置、实时状态、轨迹、故障、事故等信息进行分层叠加，实现“一图观全局”的集中监控与调度。
功能需求详细：
2.5.1.1 车辆实时位置与状态图层
• 以图标形式在地图上标注所有在线测试车辆（区分人工/自动驾驶模式图标）
• 点击车辆图标弹出信息窗，展示：
    ◦ 车辆基本信息（车牌号、VIN、所属企业、车型）
    ◦ 当前驾驶模式（自动驾驶/人工驾驶/远程遥控）
    ◦ 实时速度、航向角
    ◦ 控制模式（档位）、自动驾驶等级
    ◦ 在线状态（在线/离线）
    ◦ 最后上报时间
• 支持车辆搜索定位聚焦
2.5.1.2 行驶轨迹展示
• 选中车辆后，可叠加显示其实时行驶轨迹（近一段时间内的轨迹路径）
• 支持轨迹颜色按速度或驾驶模式渐变展示
• 支持历史轨迹回放（按时间轴拖动回放关键状态和位置）
2.5.1.3 故障与告警信息叠加
• 在地图上以醒目标识（如红色闪烁图标）标注当前存在故障或告警的车辆
• 告警车辆图标上方可显示告警数量角标
• 点击告警车辆可直接查看告警详情列表（告警类型、发生时间、处理状态）
• 支持按告警级别、类型过滤显示的车辆
2.5.1.4 事故点位标注
• 在地图上以专属图标标注已上报事故的发生位置
• 点击事故图标展示事故概要（时间、车辆、事故类型、伤亡情况等）
• 支持事故历史点位的显示/隐藏
2.5.1.5 围栏与道路状态叠加
• 叠加显示电子围栏区域（半透明多边形），不同围栏类型用不同颜色边框区分
• 叠加显示开放道路状态图层（与开放道路资源管理模块联动），以不同颜色渲染道路当前开放状态（开放/暂停/关闭）
2.5.1.6 车辆列表与地图联动
• 地图侧边栏或底部提供车辆列表视图（表格），展示关键监控字段
• 点击列表中某车辆，地图自动聚焦并高亮该车辆
• 列表支持多条件筛选（状态、企业、车辆类型等）
2.5.1.7 统计概览卡片
• 在一张图上方或悬浮区域展示核心统计数据：
    ◦ 当前在线车辆数 / 总测试车辆数
    ◦ 今日告警总数（按级别分）
    ◦ 当日事故数量
    ◦ 当前处于自动驾驶模式的车辆数
• 统计数据定时刷新
2.5.1.8 交互与性能要求
• 地图支持缩放、拖拽，缩放级别自动聚合车辆点位（用聚类展示）
• 告警车辆需保持可见（不被聚类淹没），或单独图层置顶
• 数据刷新频率：位置信息 ≥ 1Hz，告警和状态变化准实时推送
2.5.2 车辆运行监控（基础功能）
• 车端数据接入：提供统一接口（HTTP/MQTT等），车企主动推送数据；平台进行校验和入库
• 车辆视频监控：支持RTSP/RTMP等视频流协议，通过车辆数据关联视频流，在监控一张图或独立页面播放
• 位置及行驶轨迹监控：作为一张图核心数据源提供历史轨迹回放
• 自动驾驶运行监控：监控AD系统状态、等级、传感器数据、驾驶干预行为等
• 感知决策信息管理：采集传感器及决策指令数据，支持事故溯源
• 测试里程监控：基于位置数据精准统计里程
• 电池状态监控：展示电池SOC/SOH/温度等
• 故障信息监控：采集并展示故障码及描述，推送至一张图告警
2.5.3 异常状态告警
告警类型包括：
• 车辆运行异常（超速/剧烈变速/偏离路线）
• 碰撞事件
• 车载终端异常
• 临牌有效期预警
• 电子围栏越界
• 测试时段合规性
• 安全员到岗状态告警（移除排班逻辑后，改为实时检测安全员在岗信号，如安全员状态数据缺失或显示未到岗时触发）
告警信息在一张图中高亮，可通过弹窗、声音、推送等多种方式通知。
2.5.4 事故数据上报
• 测试主体/安全员通过应用端上报事故信息（时间、地点、原因等），上传佐证材料
• 监管部门审核确认。支持撤回和查询历史事故

2. 非功能需求
3.1 性能需求
• 车端推送数据接入处理延迟 ≤ 500ms
• 一张图车辆位置刷新率 ≥ 1Hz
• 告警从触发到前端展示 ≤ 3秒
• 支持至少500辆车辆同时在线监控
3.2 数据需求
• 车辆轨迹数据存储至少6个月，事故数据永久保存
• 视频数据按需存储，保留周期可配置
3.3 接口需求
• 接收车企推送的车辆数据（统一格式）
• 向开放道路模块实时同步车辆占用道路数据，并接收道路状态变更通知用于资格终止
• 与交管系统对接，接收临牌信息人工录入后的存储
3.4 安全需求
• 监控一张图仅限监管部门和授权人员访问
• 关键业务操作审计日志完整
• 视频流访问需鉴权
